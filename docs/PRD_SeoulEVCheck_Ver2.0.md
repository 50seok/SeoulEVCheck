## 0. 프로젝트 개요

**SeoulEVCheck** — 서울시 전기차 충전 데이터로 **수요를 예측하고(ML·DL), 그 예측으로 충전 운영을 권고하는(LLM)** 인텔리전스 서비스. 한 주제를 3주에 걸쳐 ML → DL → LLM으로 점진 확장한다.

### 3주 로드맵

| 주차 | 단계 | 내용 | 마감 | 핵심 기술 |
|---|---|---|---|---|
| **1주** | ML | 충전 수요 예측 (충전소·구 단위) | **6/24(수) 09:00** | XGBoost 회귀 |
| **2주** | DL | 충전 수요 시계열 예측 | 7/1 | LSTM |
| **3주** | LLM & RAG | 충전 어드바이저 (Q&A + 배치 권고) | 7/8 | RAG · LLM |

각 주차는 **독립 발표 + 독립 제출**(ipynb·py·hwp). 데이터·앱·레포는 공유하며 매주 확장한다.

### 향후 비전 (스토리)
충전 수요 예측 → 이동형 **'배달충전'** 배치 의사결정 → **자율주행 전기차 시대, 예측 수요에 따라 자율주행 충전 차량이 스스로 고수요 지역으로 이동하는 완전 무인 충전 서비스의 '두뇌'** 로 확장. (※ 자율주행은 비전이며 본 과제의 구현 범위 아님)

## 1. 사업과제

**AI 기반 서울시 전기차 충전 수요 예측 및 운영 인텔리전스 (SeoulEVCheck)**

---

## 2. 개요 및 현황

### 2.1 추진배경 및 목적
- 전기차 보급 급증으로 충전 인프라의 **수요 예측·관리** 필요성이 커지고 있다.
- 충전 수요를 지역·시간대별로 예측하면 ① 충전 인프라 투자 효율, ② 전력 부하 관리(에너지 절감), ③ 충전소 운영 최적화에 활용된다.
- **향후 비전:** 예측 수요 기반 이동형 '배달충전' 배치 → 자율주행 무인 충전 서비스의 두뇌로 확장(스토리).

### 2.2 과제 범위

| 구분 | 내용 |
|---|---|
| **AI (1·2주)** | 데이터 정제 / EDA·상관분석 / 특성공학 / 충전 수요 예측 (XGBoost·LSTM) / RMSE·R² 평가 |
| **LLM (3주)** | 예측 결과 기반 자연어 운영 권고 + 충전 정책·요금 문서 RAG Q&A |
| **시각화** | Streamlit 서비스 (구 지도 · 충전소 핫스팟 · 어드바이저 챗), 매주 확장 |
| **테스트** | 모델 검증(train/test·교차검증) + 시연 통합 테스트 |

**포함하지 않음:** 자율주행 구현, CNN/이미지 처리, 실시간 스트리밍, 동(洞) 단위 예측, 충전소 신규 입지 최적화.

### 2.3 과제 추진 방법
**1) 선정 기준** — 공공데이터(한전) 활용·63만 건 확보 / 정형 회귀라 단순·고속 / 인프라·에너지 비용 효율
**2) 적용 대상**

| 기능 | 수집 데이터 | 예측 인자(독립변수) | 예측 대상 |
|---|---|---|---|
| 전기차 충전 | 한전 서울 충전 세션 63만 건 | 구·충전기종류·충전기용량·요일·월·시간대 | 일별 충전량(kWh) — 충전소/구 |

**3) 프로세스**
```
DATA IMPORTING → 전처리 → 모델링(ML→DL) → 예측 → 어드바이저(LLM) → 시각화
한전 충전량      null·날짜    XGBoost/LSTM    수요     자연어 권고+RAG    Streamlit
```

## 3. 데이터 명세 (3주 공유)

**출처:** 한국전력공사_서울시 전기차 충전소 충전량 (공공데이터포털)

| 항목 | 값 |
|---|---|
| 행(세션) | 638,702 / 충전소 ~625 / 고유주소 611 |
| 컬럼(9) | 충전구분·충전소명·주소·충전기용량·**충전량**·충전시간·충전분·충전시작시각·충전종료시각 |
| 타깃 | 충전량(kWh) 일별 집계 |

**파생:** 주소→구(추출 98.5%), 충전시작시각→요일·월·시간대.
**품질 이슈(전처리):** null 주소 8,201세션 제거 · 오타 정제 · 날짜 분산(2014~2022, 458일)→최근 구간 필터 · 누수 컬럼(충전시간/분) 제외.

## 4. 3주 단계별 설계

### Week 1 — ML (상세, 이번 집중)
- 타깃: 충전소별 / 구별 **일 충전량** (독립 두 모델)
- 모델: XGBoost 회귀 (베이스라인·선형 → XGBoost)
- 산출: 수요 예측 + 특성중요도 + 예측 vs 실제

### Week 2 — DL (개요)
- LSTM 시계열로 일별 충전 수요 예측 (견본의 LSTM 접근 준용)
- 산출: 학습곡선 + 예측 vs 실제 시계열

### Week 3 — LLM & RAG (개요)
- **충전 어드바이저**: ① 예측 결과 → 자연어 운영 권고("강남 18시 수요↑, 배달충전 3대 권장") ② 충전 정책·요금·보조금 문서 RAG Q&A
- 산출: 챗 인터페이스 (Streamlit 통합)

## 5. Week 1 — 모델 / 평가 (상세)

| 모델 | 타깃 | 집계 행 | 용도 |
|---|---|---|---|
| A (충전소) | 충전소별 일 충전량 | 185,337 | 배달충전 핫스팟(미시) |
| B (구) | 구별 일 충전량 | 11,402 | 인프라·부하(거시) |

**실측 베이스라인 R²:** 충전소 0.582(+요일 0.600) / 구 0.757(+요일 0.765)

| 지표 | 목표 |
|---|---|
| R² | 모델 B(구) ≥ **0.80** / 모델 A(충전소) ≥ **0.65** |
| RMSE | 베이스라인 대비 개선 |

> 주 성공기준: 두 모델 모두 베이스라인 초과 + 모델 B R² ≥ 0.80. 검증: train/test 8:2 + 5-fold, `random_state=42`.

## 8. 수용 기준 (Week 1, Given-When-Then)
- [ ] Given 63만 행, When 전처리, Then null·범위밖 제거 + 구/요일/시간대 생성
- [ ] Given 두 단위 집계, Then 충전소-day·구-day 데이터셋 생성
- [ ] Given XGBoost 학습, Then 두 모델 모두 베이스라인 초과
- [ ] Given 모델 B(구), Then 테스트 R² ≥ 0.80
- [ ] Given 앱 입력, Then 예상 충전량·핫스팟 표시
- [ ] Given 배포, Then 공개 링크 시연 가능

## 9. 검증 계획

## 10. 리스크
- 충전소 단위 노이즈 → R² 미달 가능. 대응: 목표 0.65 현실화 + 핫스팟 "랭킹"은 안정.
- 날짜 분산 → 최근 구간 필터.
- 3주 일정 → 데이터·앱·유틸 공유로 작업량 관리, 매주 태그로 동결.
